# Реестр технического долга

> Единое место, куда стекается весь технический долг из завершённых Exec Plans.
> Этот файл входит в иммутабельное ядро (см. `docs/index.md`, секция B.1).

## Как работать с этим файлом

1. **Добавление:** При закрытии Exec Plan перенеси сюда пункты из секции `Leftover Tech Debt`.
2. **Приоритизация:** Человек периодически пересматривает таблицу и расставляет приоритеты.
3. **Закрытие:** Когда долг исправлен — установи статус `Done` и укажи ссылку на PR/план.

## Приоритеты

| Уровень | Значение |
|---|---|
| `Critical` | Блокирует другие задачи или вызывает баги в проде |
| `High` | Существенно замедляет разработку или ухудшает качество |
| `Medium` | Неудобство, но не блокирует |
| `Low` | Косметика, рефакторинг «когда будет время» |
