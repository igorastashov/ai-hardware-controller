# Turntable Agent Tooling MVP

## Проблема
Агенту нужен безопасный и предсказуемый способ управлять поворотным столом по BLE без риска рассинхрона, скрытых очередей и неструктурированных ошибок.

## Решение
Дать агенту фиксированный набор tool-ручек (`state`, `home`, `move_to`, `stop`) с единым JSON-контрактом, где движение сериализуется программно (single-flight), а завершение команды определяется безопасной timing-моделью до появления надежного done-сигнала от устройства.

## Критерии приёмки
- [ ] Агент может запросить текущее состояние стола через `turntable_state`.
- [ ] Агент может выполнить программный `home` через `turntable_home`.
- [ ] Агент может вызвать `turntable_move_to(rotation_deg, tilt_deg)` и получить структурированный результат.
- [ ] При параллельном вызове движения возвращается структурированная ошибка `DEVICE_BUSY` с кодом конфликта.
- [ ] `turntable_stop` доступен как аварийная команда и возвращает структурированный ACK-ответ.
- [ ] Ошибки во всех ручках возвращаются в JSON-формате (`code`, `message`, `http_status`).

## Ограничения
- В MVP не делаем автокалибровку абсолютной физической позы.
- В MVP не считаем `+DATA` эквивалентом живого энкодера.
- В MVP не используем встроенную очередь устройства как источник сериализации команд.
- В MVP не открываем публичный сетевой API (LAN bridge) до финализации runtime-политики.
