# Каталог продуктовых спецификаций

> Здесь перечислены все продуктовые требования: что делает продукт и для кого.
> Product Spec описывает **пользовательскую ценность**, а не техническую реализацию.

## Когда создавать Product Spec

- Перед началом работы над новой пользовательской фичей.
- Когда нужно зафиксировать требования для агента, чтобы он понимал «что» делать, а не только «как».

## Рекомендуемая структура Product Spec

```markdown
# [Название фичи]

## Проблема
Какую проблему пользователя мы решаем.

## Решение
Как продукт решает эту проблему (без технических деталей).

## Критерии приёмки
- [ ] Пользователь может ...
- [ ] Система должна ...

## Ограничения
Что мы НЕ делаем в рамках этой фичи.
```
